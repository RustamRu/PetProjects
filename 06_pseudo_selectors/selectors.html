CSS селекторы

Селектор - это часть CSS-правила, которая сообщает браузеру, к какому элементу (или элементам) веб-страницы будет применён стиль.

Селектор типа

Селектор типа используется для выборки элементов по их тегу и выбирает все элементы на странице с данным значением тега. Обратите внимание, что селектор типа выбирает все теги на странице, которые ему соответствуют, и соответствующим образом их стилизует

Пример

Все заголовки h4 выделены зеленым, так как использован селектор типа (h4).

Селектор класса

Класс — одно из базовых понятий в CSS. Класс прописывается в атрибуте class у элемента. Это идентификатор, т. е. произвольное имя, состоящее из латинских символов или цифр. Могут содержать в себе символ дефиса и подчеркивания. Недопустимо использование пробелов и других непечатаемых символов. Так же необходимо, чтобы первый символ был не цифрой.

После того, как элементам был присвоен класс ( с помощью атрибута class ), их можно выбрать в CSS обращением через точку (.).

Например: .description {}

На странице может присутствовать несколько элементов с одинаковым классом. В этом случае стилевые правила будут применены ко всем элементам.

Пример

Применение нескольких классов к элементу

Для одного элемента допустимо указывать несколько классов в атрибуте class.

Например: <p class="description emphasized-description">….</p>

В этом случае классы перечисляются в атрибуте через пробел. А элементу присваиваются все стили с обоих классов.

Если в классах, присвоенных элементу, присутствуют одинаковые стилевые правила, то срабатывает правило в соответствии со специфичностью (следующее занятие). Пока можем считать, что сработает то правило, которое идет ниже по каскаду css. (Вспомните приоритет стилей при разных типах подключения, и приоритет стилей в одном файле в зависимости от близости к концу документа).

Пример

Среди заголовков h4 только заголовок с двумя указанными классами "header_blue header_underline" будет выделен синим и нижним подчеркиванием.

CSS идентификатор

Вторым базовым понятием является идентификатор. Это так же произвольное имя, присваиваемое элементу, но уже в атрибуте id. Главное отличие идентификатора от класса заключается в том, что на HTML странице не допустимо использовать два или более одинаковых идентификаторов для разных элементов. Т.е. идентификатор — это уникальное значение на HTML странице, которое так же может быть использовано в якорных ссылках для перехода к этим элементам. Обращение к элементу с идентификатором id осуществляется в css через решетку (#).

Если на странице будет указано более одного идентификатора, стили применяются ко всем идентификаторам. Но это будет неверно с технической точки зрения. И может вызвать серьезные осложнения при работе с этими элементами из JavaScript. Поведение JS программы в этом случае непредсказуемо.

Так же, в отличие от классов, мы не можем присвоить несколько id одновременно на один элемент. В этом случае ни один идентификатор не сработает.

При смешении классов и идентификаторов, правила будут отрабатываться в соответствии со специфичностью (следующее занятие). Пока достаточно запомнить, что id будет иметь приоритет на классами.

Например: <p id="mistake" class="description">…..</p>

Заголовок c id1 и id2 не будет выделен ни подчеркниванием, ни красным цветом, так как указано более одного идентификатора.

Селектор потомков

До этого мы изучили с вами так называемые простые селекторы. Это селектор типа, селектор класса и селектор идентификатора. Как вы обратили внимание, такие селекторы записываются в css одним ключевым словом (например: h2, .description или #idea).

Существуют так же составные селекторы, состоящие из нескольких простых и соединенных между собой каким-то символом.

Селектор потомков — это составной селектор, в котором простые селекторы соединены через пробел. При использовании селектора потомков будут выбраны те элементы, у которых расположение по иерархии в HTML документе (DOM дереве) соответствует данному селектору.

Например: .menu li a {} - выберет все ссылки, которые вложены в элементы не маркированного списка, которые в свою очередь вложены в элементы (произвольные) с классом class="menu"

Составные селекторы пока удобно «читать» справа налево. С опытом вы будете понимать их «на-автомате»

Для селекторов потомков не имеет значения уровень вложенности элементов. Посмотрите опорный код проекта. Есть два немаркированных списка, вложенных в элемент с классом menu

В первом случае — элемент li является прямым потомком элемента ul с классом menu. И элемент a является прямым потомком элемента li.

Но если мы уберем класс menu из элемента ul на уровень выше (где мы обернули разметку в дополнительный div и присвоили класс menu этому div) - li перестанет быть прямым потомком элемента с классом menu. Теперь li вложен в .menu через элемент ul (у которого класса нет). Но стилевое правило .menu li a {} - все равно сработает.

В блоке с классом "block02" будут выделены красным все вложенные элементы p вне зависимости от уровня вложенности.

Дочерний селектор

В дочернем селекторе простые селекторы соединены через символ >

Выборка происходит так же по иерархии. Но правила срабатывают только на тех элементах, которые непосредственно вложены в родительские.

Например: .menu2 > li a

Сработает на всех элементах a, которые вложены в элементы li (причем вне зависимости от уровня вложенности). Но элементы li должны быть прямыми потомками элементов с классом menu2

Пример

В блоке с классом "block03" будут выделены красным все вложенные элементы p только на первом уровне вложенности.

Общий селектор смежных элементов

Он похож на сестринский селектор. Но осуществляет более широкую выборку элементов, которая осуществляется для всех элементов на одном уровне иерархии. Записывается с помощью символа тильда (~)

Например:

h2 ~ .description {}, сработает для всех элементов с классом description, которые находятся на одном уровне иерархии с h2

h2+.description {} сработает только для .description, который идет сразу за h2

Группирующий селектор

Для облегчения поддержки (обслуживания кода), а так же экономии времени труда программиста, возможно, объединить одинаковые стилевые правила под группирующим селектором. Группирующий селектор - это перечень селекторов (как простых, так и составных), к которым будут применены одинаковые правила.

Например:

h2,
h3 {
text-transform: lowercase;
}

все заголовки h2 и h3 будут выведены в нижнем регистре.

Селектор атрибутов

Очень похож на рассмотренные ранее простые селекторы. Но при этом дополнительно осуществляется проверка на равенство атрибута некоторому значению. Данный селектор удобно применять для разметки элементов формы, чтобы делать выборку по атрибуту type у элементов input, тем самым стилизуя, применяя стилевые правила к разным элементам ввода

Например:

input[type="text"] {
width: 150px;
display: block;
margin-bottom: 10px;
background-color: yellow;
}

input[type="button"] {
background-color: cyan;
}

Также допустимо обращаться к любым другим атрибутам. И использовать не только селектор типа (как основной), но и другие простые селекторы.

Например:

.button[type="textarea"] {
border: 1px solid blue;
}

.button[data-foo="bazz"] {
border: 1px solid green;
}
